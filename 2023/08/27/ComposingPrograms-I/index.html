Chapter1: Building Abstractions with Functions

1.1 Getting Started

1.2 Elements of Programming

  • 1.2 Elements of Programming
  • Every powerful language has three mechanisms:
    • primitive expressions and statements, which represent the simplest building blocks that the language provides,
    • means of combination, by which compound elements are built from simpler ones, and
    • means of abstraction, by which compound elements can be named and manipulated as units.

1.2.1 Expressions

  • infix notation: a way of writing mathematical expressions where the operator is placed between the operands

1.2.2 Call Expressions

  • Some key characteristics of call expressions in Python:
    • The function name comes first, followed by opening and closing parentheses. For example: my_function()
    • Arguments are passed inside the parentheses separated by commas. For example: my_function(arg1, arg2)
    • Arguments can include literal values, variables, expressions, etc. For example: my_function(5, x, y+1)
    • Calling functions executes the code in the function body using the passed arguments.
    • The return value from the function is used where the call expression appears. For example: print(my_function(5))
    • Call expressions can be nested, like func1(func2(arg))
    • The number of arguments must match the function definition, unless default values are specified.
    • Keyword arguments can be used with call expressions for clarity. For example: my_function(arg1=5, arg2=x)

1.2.3 Importing Library Functions

  • Python organizes the functions and other quantities that it knows about into modules, which together comprise the Python Library. To use these elements, one imports them.

1.2.4 Names and Environment

  • assignment operator = allows us to use names to refer to computational objects

  • names are also bound via import statements

  • Python evaluates the expression to the right of = before changing the binding to the name on the left.

  • Changing the value of one name does not affect other names.

  • An environment is a mapping of names to objects. For example, a function has its own local environment that maps the names of its parameters to objects.

  • Each environment has its own lifetime - for example, local environments are created when a function is called and destroyed when it returns.

1.2.5 Evaluating Nested Expressions

  • To evaluate a call expression, Python will do the following:
    1. Evaluate the operator and operand subexpressions, then
    2. Apply the function that is the value of the operator subexpression to the arguments that are the values of the operand subexpressions.

1.2.6 The Non-Pure Print Function

  • Pure functions. Functions have some input (their arguments) and return some output (the result of applying them). Pure functions have the property that applying them has no effects beyond returning a value. Moreover, a pure function must always return the same value when called twice with the same arguments.
  • Non-pure functions. In addition to returning a value, applying a non-pure function can generate side effects, which make some change to the state of the interpreter or computer. A common side effect is to generate additional output beyond the return value, using the print function.
